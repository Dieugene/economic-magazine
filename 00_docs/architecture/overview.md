# Архитектура проекта: Фронтенд журнала «Вопросы теоретической экономики»

## Общая структура

Система состоит из двух фронтенд-приложений, работающих с единым REST API бэкенда:

```
┌─────────────────────────────────────────────────────┐
│                    Фронтенд                         │
│                                                     │
│  ┌──────────────────┐    ┌───────────────────────┐  │
│  │  Публичный сайт  │    │      Админ-панель      │  │
│  │   (SSG/SSR)      │    │       (SPA/SSR)        │  │
│  │                  │    │                        │  │
│  │ - Главная        │    │ - Управление номерами  │  │
│  │ - Архив номеров  │    │ - Карточки статей      │  │
│  │ - Страница статьи│    │ - Загрузка PDF/обложек │  │
│  │ - Рубрикатор     │    │ - Публикация           │  │
│  │ - Редколлегия    │    │                        │  │
│  │ - Авторам        │    │                        │  │
│  │ - Поиск          │    │                        │  │
│  └────────┬─────────┘    └───────────┬────────────┘  │
│           │                          │               │
└───────────┼──────────────────────────┼───────────────┘
            │         REST API         │
            │        (JSON)            │
┌───────────┴──────────────────────────┴───────────────┐
│                                                      │
│           Бэкенд (Django, не наша часть)              │
│           - Хранение метаданных                      │
│           - Хранение/отдача PDF-файлов               │
│           - Авторизация                              │
│                                                      │
└──────────────────────────────────────────────────────┘
```

## Ключевые компоненты

### Публичный сайт

Публичная часть журнала, доступная всем посетителям. Сохраняет структуру текущего сайта questionset.ru, но с современным дизайном.

**Страницы:**

| Страница | Описание | Источник данных |
|----------|----------|-----------------|
| Главная / О журнале | Информация о журнале, обращение редактора, ВАК, ISSN | Статический контент + API (свежий номер) |
| Архив по годам | Сетка обложек номеров за выбранный год | API: список номеров по году |
| Страница номера | Содержание: статьи сгруппированы по рубрикам, PDF полного выпуска | API: номер с вложенными статьями |
| Страница статьи | Метаданные, аннотация, ссылки на PDF и XML | API: статья по ID |
| Рубрикатор | Список статей по тематической рубрике за все годы | API: статьи по рубрике |
| Редколлегия | Список членов с регалиями, ORCID, SPIN | API или статический контент |
| Авторам | Порядок подачи, авторское соглашение, ссылка на ms.questionset.ru | Статический контент |
| Этика публикаций | Информационная страница | Статический контент |
| Контакты | Контактная информация редакции | Статический контент |
| Поиск | Поиск по названиям статей, авторам, ключевым словам | API: поисковый эндпоинт |

**Двуязычность:** русская и английская версии. Переключатель языка в шапке. Контент статей хранится в двух языковых вариантах (название, аннотация, ключевые слова, сведения об авторах).

**SEO:** Статическая генерация публичных страниц (SSG) с инкрементальной регенерацией при публикации нового номера. Метатеги, OpenGraph, структурированные данные (schema.org/ScholarlyArticle).

### Админ-панель

Закрытый кабинет для ответственного пользователя (ответственный секретарь / главный редактор). Простая авторизация — логин/пароль, без системы ролей.

**Функциональные блоки:**

**1. Управление номерами**
- Список всех номеров журнала (таблица с фильтрацией по году, статусу)
- Создание нового номера: год, порядковый номер, сквозной номер, дата выхода
- Статусы номера: черновик → готов к публикации → опубликован
- Загрузка обложки (изображение) и PDF полного выпуска

**2. Управление статьями номера**
- Список статей в номере (с drag-and-drop сортировкой)
- Карточка статьи — форма со следующими полями:

| Группа | Поля |
|--------|------|
| Основное | Рубрика (выбор из 7), тип (научная/обзор/рецензия/от редактора), страницы (от-до) |
| Заглавие | Название на русском, название на английском |
| Авторы | Список авторов, для каждого: ФИО (рус/англ), аффилиация (рус/англ), email, ORCID |
| Аннотация | Текст на русском, текст на английском |
| Ключевые слова | На русском, на английском |
| Идентификаторы | DOI, УДК, JEL-коды |
| Библиография | Список источников (ЛИТЕРАТУРА), список источников (REFERENCES) |
| Даты | Дата получения, дата принятия в печать |
| Финансирование | Источники финансирования (текст) |
| Файлы | PDF статьи (загрузка), ссылка на XML (JATS, внешняя ссылка на РЦНИ) |

**3. Предпросмотр и публикация**
- Предпросмотр страницы статьи (как будет выглядеть на сайте)
- Предпросмотр содержания номера
- Кнопка «Опубликовать номер» — делает номер доступным на публичном сайте

### Общие UI-компоненты

- **Шапка (Header):** логотип, название журнала, ISSN, навигация, поиск, переключатель языка
- **Навигация:** адаптивное меню с выпадающими подменю (О журнале, Редколлегия, Рубрикатор, Архив, Авторам, Этика)
- **Хлебные крошки (Breadcrumbs):** на всех страницах
- **Боковая панель:** блок «Свежий номер» (обложка + дата + ссылка)
- **Подвал (Footer):** ссылки на основные разделы, копирайт, учредитель, политика персональных данных
- **Карточка статьи:** переиспользуемый компонент для отображения статьи в списках (автор, название, страницы, DOI, ссылка на PDF)

## Принципы взаимодействия

### Фронтенд ↔ Бэкенд

- Взаимодействие через REST API (JSON)
- Аутентификация админки: JWT-токены (или сессии Django — на усмотрение бэкенда)
- Загрузка файлов (PDF, изображения): multipart/form-data на соответствующие эндпоинты бэкенда
- PDF-файлы хранятся на сервере бэкенда и отдаются по прямым ссылкам
- Swagger-спецификация согласуется между фронтендом и бэкендом; фронтенд может предложить встречную спецификацию

### Основные API-сущности (предварительно)

```
GET  /api/issues/                     # Список номеров (фильтр по году)
GET  /api/issues/{id}/                # Номер с содержанием
GET  /api/articles/{id}/              # Статья с полными метаданными
GET  /api/articles/?section={slug}    # Статьи по рубрике
GET  /api/articles/search/?q={query}  # Поиск
GET  /api/editorial-board/            # Редколлегия
GET  /api/pages/{slug}/               # Статические страницы

POST /api/admin/issues/               # Создание номера
PUT  /api/admin/issues/{id}/          # Обновление номера
POST /api/admin/issues/{id}/publish/  # Публикация номера
POST /api/admin/articles/             # Создание статьи
PUT  /api/admin/articles/{id}/        # Обновление статьи
POST /api/admin/upload/               # Загрузка файлов
```

### Внешние системы

| Система | Взаимодействие |
|---------|---------------|
| ms.questionset.ru | Ссылка на внешний сервис подачи статей (не интегрируем) |
| journals.rcsi.science (РЦНИ) | Внешняя ссылка на JATS XML — просто URL в карточке статьи |
| elibrary.ru | Загрузка метаданных делается отдельно, не через наш сайт |

## Технологический стек

| Компонент | Технология | Обоснование |
|-----------|------------|-------------|
| Фреймворк | **Next.js 15 (React 19)** | SSG/SSR для SEO публичной части; серверные компоненты для производительности; API routes при необходимости; развитая экосистема UI-компонентов для админки |
| Язык | **TypeScript** | Типизация API-контрактов, надёжность, автодокументация |
| Стилизация | **Tailwind CSS** | Быстрая разработка, консистентный дизайн, хорошая поддержка адаптивности |
| UI-компоненты | **shadcn/ui** | Качественные доступные компоненты (формы, таблицы, модалки), легко кастомизируемые |
| Формы (админка) | **React Hook Form + Zod** | Валидация форм с карточками статей, типобезопасность |
| Работа с API | **SWR или TanStack Query** | Кэширование, ревалидация, оптимистичные обновления |
| i18n | **next-intl** | Интернационализация (RU/EN) с поддержкой SSG |
| Иконки | **Lucide React** | Легковесный набор иконок |
| Линтинг | **ESLint + Prettier** | Единый стиль кода |

### Обоснование выбора Next.js

- **SSG с инкрементальной регенерацией (ISR):** Публичные страницы генерируются статически при сборке и обновляются при публикации нового номера. Это даёт мгновенную загрузку и отличный SEO — критически важно для научного журнала
- **Серверные компоненты (React Server Components):** Уменьшают размер JS-бандла для публичной части — посетителю не нужно загружать React для чтения статической страницы
- **Единая кодовая база:** Публичный сайт и админка в одном проекте, с общими типами и компонентами
- **Middleware:** Простая защита админских роутов через middleware (проверка JWT)

## Ограничения

### Технические
- Бэкенд разрабатывается отдельной командой (Владимир); API-контракт согласуется через Swagger
- PDF-файлы хранятся на стороне бэкенда; фронтенд только ссылается на них
- JATS XML генерируется внешним сервисом РЦНИ; фронтенд хранит только ссылку
- Сервис подачи статей (ms.questionset.ru) остаётся внешним — интеграция не планируется

### Бизнесовые
- Целевая аудитория — люди с гуманитарным образованием; интерфейс должен быть максимально простым и понятным
- Привычная структура навигации сохраняется; радикальные изменения UX нежелательны
- Поиск по авторам и ключевым словам желателен, но не является обязательным требованием ВАК
- Функционал админки значительно проще, чем у системы «Артикулюс» (elibrary.ru) — только то, что нужно для одного журнала
